# Настройка автоматической отправки опросов

## Проблема
Бесплатный план Render усыпляет сервис после 15 минут неактивности. Из-за этого встроенное расписание (job_queue) не работает, если бот спит.

## Решение
Используем внешний бесплатный сервис **cron-job.org**, который будет "будить" бота и отправлять опрос каждую среду в 10:30 (Алматы время).

## Инструкция по настройке

### Шаг 1: Деплой бота на Render
1. Запушьте обновленный код на GitHub
2. Разверните/обновите бот на Render.com
3. Скопируйте URL вашего сервиса (например: `https://doptep-poll-bot.onrender.com`)

### Шаг 2: Регистрация на cron-job.org
1. Перейдите на сайт: https://console.cron-job.org/signup
2. Зарегистрируйтесь (бесплатный аккаунт)
3. Подтвердите email

### Шаг 3: Создание Cron Job
1. Войдите в панель управления: https://console.cron-job.org/jobs
2. Нажмите **"Create cronjob"**
3. Заполните форму:

   **Title (название):**
   ```
   DopTep Wednesday Poll
   ```

   **Address (URL):**
   ```
   https://ваш-домен.onrender.com/trigger-poll
   ```
   Замените `ваш-домен.onrender.com` на ваш реальный URL от Render.

   **Schedule type:**
   - Выберите **"Every week"**

   **Day of week:**
   - Выберите **Wednesday (среда)**

   **Execution time:**
   - Время: **04:30** (это 10:30 по времени Алматы, так как cron-job.org использует UTC)
   - Чтобы рассчитать: 10:30 Алматы (UTC+5) = 04:30 UTC

   **Request method:**
   - Выберите **GET**

   **Request timeout:**
   - Оставьте по умолчанию (30 секунд)

4. Нажмите **"Create cronjob"**

### Шаг 4: Дополнительный Health Check (опционально)
Чтобы бот не засыпал в течение дня, можно настроить дополнительный cron job:

1. Создайте еще один cronjob
2. **Title:** `DopTep Keep Alive`
3. **Address:** `https://ваш-домен.onrender.com/health`
4. **Schedule:** Every 10 minutes (каждые 10 минут)
5. **Request method:** GET

Это будет держать бот активным большую часть времени.

## Как работает

1. **Каждую среду в 04:30 UTC (10:30 Алматы)**
   - cron-job.org отправляет GET запрос на `/trigger-poll`
   - Если бот спит, Render его "будит" (занимает ~30 секунд)
   - Endpoint `/trigger-poll` запускает функцию `send_poll()`
   - Опрос и список игроков отправляются в группу

2. **Health check (если настроен)**
   - Каждые 10 минут проверяет `/health`
   - Держит бот активным, чтобы не засыпал

## Проверка работы

### Тест вручную
Откройте в браузере:
```
https://ваш-домен.onrender.com/trigger-poll
```

Должно отобразиться: **"Poll sent successfully!"**

### Просмотр логов
1. На cron-job.org: во вкладке **"History"** можно увидеть историю выполнения
2. На Render.com: в разделе **"Logs"** можно увидеть логи бота

## Альтернативные сервисы

Если cron-job.org не подходит, можно использовать:
- **UptimeRobot** (https://uptimerobot.com/) - мониторинг + HTTP запросы
- **EasyCron** (https://www.easycron.com/) - аналог cron-job.org
- **GitHub Actions** - бесплатные workflows с расписанием

## Важно

- **Часовой пояс:** cron-job.org использует UTC. Алматы = UTC+5
  - 10:30 Алматы = 04:30 UTC
  - 11:00 Алматы = 05:00 UTC

- **Render холодный старт:** Первый запрос после сна может занять 30-60 секунд

- **Бесплатный план Render:** Ограничение 750 часов в месяц. Health check каждые 10 минут использует больше времени, но должно хватить.

## Тестирование

Для теста можно временно установить время на ближайшую минуту:
1. Узнайте текущее время UTC: https://time.is/UTC
2. Установите время на cron-job.org на 2-3 минуты вперед
3. Дождитесь выполнения
4. Проверьте группу в Telegram
5. Верните правильное расписание (среда 04:30 UTC)

## Контакты для поддержки
Если возникли проблемы:
- Проверьте логи на Render.com
